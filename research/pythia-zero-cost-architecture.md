# Pythia Zero-Cost Architecture

> **Generated by:** Pythia Oracle Engine (daemon `gd_mmkryrm1_3`, session `pythia-engine-v4`)
> **Date:** 2026-03-10
> **Design Constraint:** $0.00/month, zero API keys, zero privacy leakage, open-source adoptable

---

## The $0/month Stack

### Layer 1: Data Plane (LCS — 100% Local)
- **Parsing:** tree-sitter (native Node.js bindings)
- **Embeddings:** @xenova/transformers running all-MiniLM-L6-v2 via ONNX in Node.js process
  - 384 dimensions, ~90MB model
  - Zero API calls, zero data leaves machine
- **Storage:** sqlite-vec (SQLite + vector extension) or LanceDB (embedded)
  - Both vectors and graph adjacency in local files

### Layer 2: Interface Plane (MCP)
- Single MCP server exposing both LCS and Pythia tools
- 8 tools total (down from 13+)

### Layer 3: Cognition Plane (Pythia — Free CLI)
- Gemini CLI via child_process.spawn()
- LCS fetches 3K targeted tokens → passes to CLI with question
- $0.00 (CLI bypasses API rate limits)

### Layer 4: Memory Plane (Pythia State)
- Episodic checkpoints as JSON in SQLite
- Linked to LCS graph node IDs
- Atomic writes, OS-level file locks

## Embedding Model Comparison

| Model | MTEB Score | Dimensions | Cost | Runs In |
|-------|-----------|------------|------|---------|
| all-MiniLM-L6-v2 (ONNX) | ~58% | 384 | $0 local | Node.js process |
| nomic-embed-text v1 (Ollama) | ~62% | 768 | $0 local | Requires Ollama daemon |
| nomic-embed-code (Ollama) | Beats OpenAI on CodeSearchNet | 768 | $0 local | Requires Ollama (7B, heavy) |
| OpenAI text-embedding-3-small | ~62% | 1536 | $0.02/1M tok | API call (privacy risk) |
| Gemini embedding-001 (AI Studio) | ~63% | 768 | $0 free tier | API call (Google trains on input) |

**Winner for open-source:** all-MiniLM-L6-v2 via ONNX — no dependencies, no daemons, no API keys

**Why quality gap doesn't matter:** tree-sitter AST chunking produces perfectly bounded, semantically whole code chunks. The embedding just acts as a fuzzy matcher on already-perfect chunks. Small model + perfect chunking ≈ large model + naive chunking.

---

## What We Sacrifice vs $11/month SDK

1. Embedding nuance (384d vs 1536d) — mitigated by AST chunking
2. Indexing speed (CPU minutes vs API seconds) — background task, doesn't block user
3. CLI statefulness (process-per-turn) — accepted trade-off for $0 cost

## What We Gain
- Zero friction adoption: `npm install` and go
- Zero privacy leakage: all code stays local
- Zero billing setup: no API keys, no credit cards
- Zero vendor lock-in: works offline after model download

## Tiered Model Strategy (Optional)

For adopters who WANT better quality and are willing to pay/install:

```
Tier 0 (Default):  ONNX all-MiniLM-L6-v2     — $0, no deps
Tier 1 (Ollama):   nomic-embed-text            — $0, requires Ollama
Tier 2 (API):      OpenAI text-embedding-3-small — $0.02/1M tokens
Tier 3 (API):      Voyage AI code-2             — best quality, paid
```

Users configure embedding backend in pythia.config.json. Default = Tier 0.

## GCP / Cloud Options (Not Recommended for Default)

- Vertex AI: No always-free tier. Requires GCP project + billing + service accounts.
- Gemini AI Studio embeddings: Free but Google trains on your input. Privacy blocker.
- Both violate the "frictionless adoption" constraint.
