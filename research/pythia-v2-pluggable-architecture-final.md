## Four Provider Interfaces

```typescript
export interface EmbeddingProvider {
  embedBatch(chunks: string[]): Promise<number[][]>;
  getDimensions(): number;
  getModelName(): string;
}

export interface VectorStore {
  init(dimensions: number): Promise<void>;
  upsert(items: { id: string; text: string; vector: number[]; metadata: any }[]): Promise<void>;
  search(queryVector: number[], limit: number): Promise<{ id: string; text: string; score: number }[]>;
}

export interface GraphStore {
  init(): Promise<void>;
  upsertEdges(edges: { sourceId: string; targetId: string; type: string }[]): Promise<void>;
  getNeighbors(nodeId: string, depth: number, edgeTypes?: string[]): Promise<string[]>;
}

export interface ReasoningProvider {
  spawn(sessionName: string, systemPrompt: string): Promise<string>;
  ask(sessionId: string, prompt: string): Promise<string>;
  dismiss(sessionId: string, hard: boolean): Promise<void>;
}
```

## Default Stack ($0, zero-config)

| Layer | Provider | Tech | Storage | Cost |
|-------|----------|------|---------|------|
| Embeddings | OnnxEmbeddingProvider | @xenova/transformers + all-MiniLM-L6-v2 | In-process | $0 |
| Vectors | SqliteVecStore | sqlite3 + sqlite-vec extension | .pythia/lcs.db | $0 |
| Graph | SqliteGraphStore | Adjacency tables (Source, Target, Type) | .pythia/lcs.db (same file) | $0 |
| Reasoning | GeminiCliProvider | child_process.spawn("gemini") | CLI OAuth | $0 |

**User experience:** `npm install -g pythia-engine` → `pythia index` → `pythia ask` → works.

## Our Stack (Maximum Quality)

| Layer | Provider | Tech | Storage | Cost |
|-------|----------|------|---------|------|
| Embeddings | VoyageProvider | voyage-code-3 (code-specific) | API | ~$0.02/1M tok |
| Vectors | LanceDbStore | LanceDB (columnar, fast metadata filtering) | .pythia/vectors.lance | $0 |
| Graph | KuzuStore | Kuzu (embedded graph DB, Cypher queries) | .pythia/graph.kuzu | $0 |
| Reasoning | GeminiSdkProvider | @google/genai ChatSession (gemini-1.5-pro) | API | ~$11/mo |

## Storage: One DB vs Two

| Scenario | Decision | Reason |
|----------|----------|--------|
| Default stack | ONE file (.pythia/lcs.db) | Frictionless. One file to backup. SQLite CTEs are slow but sufficient for single dev. |
| Our stack | TWO dirs (LanceDB + Kuzu) | Purpose-built engines. Atomic indexing via temp dir + rename. |

## Why This Works

- Architecture (RAG + Stateful Daemon) decoupled from Infrastructure (API vs Local)
- Same 8 MCP tools regardless of backend
- Config swap = quality upgrade, no code changes
- Default serves open-source community at $0
- Power config serves us at maximum quality
