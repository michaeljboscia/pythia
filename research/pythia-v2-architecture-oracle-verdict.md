## Why v1 Cannot Be Saved

The fatal flaw: **the daemon does not exist.** By wrapping `gemini-cli` via `executeWithFallback`, Pythia v1 simulates a persistent stateful process by spinning up a brand new ephemeral process (`gemini -r latest`) for every query. Every turn re-reads the transcript from disk and re-tokenizes the entire history. This guarantees O(N) latency scaling, explosive context pressure, and complete DPI decay vulnerability. It is not a daemon — it is an elaborate, highly fragile shell script.

Surgical refactoring cannot fix a system whose core abstraction is an illusion.

## v2 Architecture: Event-Driven & Context-Cached

### 1. The Supervisor (Erlang-Inspired)

- Single persistent Node process manages a Map of stateful worker instances
- `simple_one_for_one` strategy — each oracle is an independent worker
- MaxR/MaxT sliding window for restart throttling
- Parent watchdog: graceful `.close()` on all SDK streams + flush to disk on ppid death

### 2. The Transport (Native SDK, Not CLI)

- Bypass `gemini-cli` entirely
- Use `@google/genai` SDK via HTTP/SSE
- `ChatSession` objects hold conversation state in memory
- No re-tokenization on every turn — just `chat.sendMessage(question)`
- HTTP stream backpressure instead of node-pty XON/XOFF

### 3. The Daemon (True Stateful Worker)

- TypeScript class in MCP server memory holding active `ChatSession`
- `ask_daemon` = `chat.sendMessage()` — O(1) per turn, not O(N)
- Supervision: exception → state="dead" → halt (no ghost processes)
- SDK stream `.close()` on dismiss, not just directory deletion

### 4. Corpus Loading (Context Caching API)

1. Hash corpus with canonical normalization (strip BOM, CRLF→LF)
2. Upload to Gemini File API
3. Create Cached Context on Google servers
4. Daemon attaches to Cached Context ID at spawn
- Token cost drops 50-75%
- First query latency: 2s (vs 30s)
- Corpus mathematically guaranteed in active attention heads

### 5. Checkpoints (Knowledge Graphs, Not Markdown)

**Component A — The Graph:**
- Structured Output extraction of [Subject, Predicate, Object] triples
- e.g., `{"subject": "Supabase", "predicate": "requires", "object": "RLS Policies"}`
- Immune to semantic drift — generation 4 cannot hallucinate away a JSON property

**Component B — Session State:**
- msgpackr or Zstandard-compressed JSON of exact message array
- `[{role: 'user', parts:...}, {role: 'model', parts:...}]`
- Binary serialization, not LLM-generated prose

**Reconstitution:**
- Generation N+1 loads binary message array + immutable JSON KG as system instruction
- No degraded summary paragraph — exact replay of high-value interactions

### 6. Atomic Locking (OS-Level)

- Drop JS `async-mutex` (single-process illusion)
- POSIX directory locks (`mkdir(lock_path)`) or `flock`
- Mathematically safe across multiple Claude Code instances

### 7. Canonical CAS Hashing

- Strip 3-byte UTF-8 BOM
- Normalize CRLF→LF
- Then SHA-256
- Prevents cross-platform hash divergence (the Git 2006 lesson)
