## What We Gain (Obsidian Route)

- **Zero engineering time** — install plugin, start typing
- **World-class UI** — graph view, bidirectional linking, split panes, markdown rendering
- **The ecosystem** — Dataview, Templater, Excalidraw, hundreds of plugins
- **Local-first markdown** — portable, vendor-agnostic .md files
- **Native graph visualization** — relationships between notes rendered visually

## What We Lose (Obsidian Route)

- **The AST** — Obsidian plugins chunk text naively (split every 500 words). They slice functions in half. Tree-sitter AST chunking is mandatory for code retrieval accuracy.
- **The true graph** — Obsidian's graph = manual `[[WikiLinks]]`. It doesn't know `routes.ts` imports `auth.ts` which calls `hashPassword()`. LCS extracts this deterministically via Tree-sitter.
- **The autonomous daemon** — Obsidian is passive UI. Pythia is an active MCP participant in multi-agent orchestration. Claude Code can delegate to Pythia mid-task.
- **RAG control** — Smart Connections moved proprietary/paywalled. Locked into their embedding model and chunking choices. No RRF fusion, no cross-encoder re-ranking.

## The Middle Path: "Obsidian-as-Glass"

Don't choose between Obsidian's UI and Pythia's intelligence. Use both:

- **Obsidian is the Viewport:** Markdown editor and visualization. ADRs, checkpoints, daily notes live here.
- **Pythia+LCS is the Engine:** Background MCP server with AST-aware retrieval, graph traversal, agentic reasoning.
- **The Integration:** Pythia reads the Obsidian vault as corpus. When Pythia writes episodic memories (checkpoints), it writes `.md` files directly into an Obsidian vault folder (`/Pythia-Memories/`).

Result: Beautiful graph UI of Obsidian + deep AST-aware agentic reasoning of Pythia. Claude Code talks to Pythia via MCP, Pythia updates the Obsidian vault.

## Are We Overengineering?

| Use Case | Overengineered? |
|----------|----------------|
| "Where is the login form?" | Yes — Copilot/Smart Connections is fine |
| "Refactor auth across 12 files without breaking the ADR" | No — requires AST graph + stateful daemon |
| "What is our error handling pattern across the backend?" | No — requires graph traversal + community summaries |
| "Keep watching this refactor and flag ADR violations" | No — requires autonomous MCP agent |

## The Verdict

**Don't scrap Pythia+LCS. Don't build a UI.**

Build the engine (SQLite + ONNX + Tree-sitter + MCP). Let it write memories into an Obsidian vault. Let Obsidian be the glass. Let Pythia be the mind.
