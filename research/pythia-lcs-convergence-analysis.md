## Combined Architecture (4 Layers)

Layer 1 — DATA PLANE (LCS)
  Tech: tree-sitter, LSP, Qdrant/LanceDB, Kuzu/SQLite
  Role: Deterministic observer. Parses code, tracks AST, generates embeddings. No opinions.

Layer 2 — INTERFACE PLANE (MCP)
  Tech: Model Context Protocol Server
  Role: Exposes LCS as tools: lcs_query_graph, lcs_semantic_search, lcs_get_references

Layer 3 — COGNITION PLANE (Pythia v2)
  Tech: Node.js + native Gemini SDK ChatSession objects
  Role: Stateful reasoner. Queries LCS for context. Doesn't store corpus.

Layer 4 — MEMORY PLANE (Pythia State)
  Tech: Atomic JSON writes to filesystem
  Role: Episodic checkpoint — rationale + LCS Node ID pointers
