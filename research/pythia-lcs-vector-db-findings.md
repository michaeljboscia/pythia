## sqlite-vec: Validated as $0 Default

### What we assumed
sqlite-vec is "good enough" for small codebases.

### What the research proves
- At <100K records, brute-force k-NN delivers ~100ms latency with zero index overhead (VD-02)
- At 50K vectors / 256 dimensions: sub-20ms exact search, mathematically perfect 1.0 recall
- Single `.db` file. Zero config. Native BM25 via FTS5 in same database.
- **sqlite-vec is not just "good enough" — it's optimal at our scale**

## LanceDB: DROPPED from Premium Stack

### What we assumed
LanceDB is the premium embedded alternative — columnar format, fast metadata filtering.

### What the research proves (VD-02, VD-07)
- **Wrong tool for the job** — LanceDB is an enterprise data-lake tool (Parquet successor) built for 100M+ vectors on S3
- **Product Quantization destroys recall** — PQ compresses 128x but needs `refine_factor` disk reads to compensate
- **Throughput ceiling** — open-source version maxes at 10-50 QPS concurrent writes in Node.js process
- **RAM wall** — IVF index building fails at 128GB RAM without manual batching workarounds
- **The metaphor:** Using a cargo ship to move a couch. All overhead, none of the benefits at 50K scale.

### Decision D-68: LanceDB removed from architecture

---

## Qdrant: New Premium Stack Choice

### Why Qdrant wins at 50K vectors (VD-01, VD-07)
- **HNSW algorithm** — sub-millisecond latency at 0.99 recall for in-RAM datasets
- **Filterable HNSW** — filter by metadata (`file_path`, `artifact_type`) without search graph degradation. Solves our exact GraphRAG filtering problem.
- **Native Hybrid Search** — server-side RRF fusion of sparse (BM25) + dense (vector). We hand it both, it does the math.
- **Memory footprint** — 50K vectors × 256d = ~100MB RAM. Trivial for desktop.
- **Scales gracefully** — if we throw 50 repositories at it simultaneously, it handles it. sqlite-vec won't.
- **Install:** Docker or Homebrew binary

### Decision D-69: Premium VectorStore is Qdrant (replaces LanceDB)

## Revised Storage Architecture

### Default Stack ($0, frictionless, `npm install`)

| Component | Tech | Why |
|-----------|------|-----|
| Vectors | sqlite-vec | Brute-force exact search, <20ms at 50K/256d, perfect 1.0 recall, zero config |
| Sparse/BM25 | SQLite FTS5 | Same database file, native full-text search |
| Graph | SQLite adjacency tables | Same database file, recursive CTEs |
| **Total** | **ONE file: `.pythia/lcs.db`** | **$0, zero dependencies** |

### Premium Stack (max quality, Docker/Brew)

| Component | Tech | Why |
|-----------|------|-----|
| Vectors | Qdrant | Filterable HNSW, native RRF hybrid search, sub-ms latency |
| Sparse/BM25 | Qdrant (native) | Server-side sparse-dense fusion |
| Graph | Kuzu (pending GD-01 research) | Cypher queries, embedded C++ engine |
| **Total** | **Two services** | **Max quality, scales to multi-repo** |
